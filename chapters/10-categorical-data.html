10.1 分类结果也来自明确的观测单位

前面两章处理的是连续响应变量（虫口密度、开花比例）。但保护研究中大量结果是分类的：树木是否受害、动物是否被检出、幼苗是否成活。分类数据的分析有自己的一套逻辑：起点仍然是观测单位。

本章的主项目回到森林病虫害试验。我们关心的不是虫口密度的均值，而是：综合防治是否降低了树木受害（damage_class ≥ 3）的比例？

在开始分析之前，先确认： - 事件定义：“受害” = damage_class ≥ 3（明确阈值，可复核） - 观测单位：树木（但树木嵌套于小区：聚集问题见 10.6 节） - 分母：每个处理小区的树木总数

forest <- read.csv("../data/teaching/forest_pest_trial_long.csv",
                   stringsAsFactors = FALSE)

# 第三次调查
f3 <- forest[forest$visit == 3, ]

# 定义受害事件
f3$damaged <- f3$damage_class >= 3

# 先看小区层级的受害比例
plot_damage <- aggregate(
  damaged ~ block_id + plot_id + treatment,
  data = f3,
  FUN = function(x) c(n = length(x), damaged = sum(x))
)
plot_damage$n <- plot_damage$damaged[, "n"]
plot_damage$damaged_count <- plot_damage$damaged[, "damaged"]
plot_damage$damaged <- NULL
plot_damage$prop <- plot_damage$damaged_count / plot_damage$n
head(plot_damage)
定义事件互斥且可复核
保留分母分子和分母同等重要
检查独立卡方与 Fisher
估计效应风险差/风险比/优势比
沟通风险基线 + 绝对差

学完本章后，你能够：

  1. 定义互斥且可复核的分类结果，并正确识别分子和分母。
  2. 在卡方检验和 Fisher 精确检验之间做出有依据的选择。
  3. 区分风险差、风险比和优势比，并解释为什么优势比不等于风险比。
  4. 识别聚集数据中的伪重复问题：树木嵌套于小区。
  5. 在报告风险时同时给出基线风险和绝对差异。

10.2 比例必须同时保留分子和分母

“受害率 60%”。这个数字丢失了最关键的信息：分母是多少？

  • 3/5 = 60%：只有 5 棵树，随机波动极大
  • 60/100 = 60%：100 棵树，估计相对稳定
  • 600/1000 = 60%：1000 棵树，估计很精确

同样的比例，完全不同的证据强度。永远不要只报告比例而不报告分子和分母。

# 演示：相同比例，不同信息量
# 这三个"60%"的证据强度完全不同
cat("3/5 =", round(100 * 3/5), "%  ：95% CI: [",
    round(binom.test(3, 5)$conf.int[1] * 100), "%, ",
    round(binom.test(3, 5)$conf.int[2] * 100), "%]\n")
cat("60/100 =", round(100 * 60/100), "%  ：95% CI: [",
    round(binom.test(60, 100)$conf.int[1] * 100), "%, ",
    round(binom.test(60, 100)$conf.int[2] * 100), "%]\n")
cat("600/1000 =", round(100 * 600/1000), "%  ：95% CI: [",
    round(binom.test(600, 1000)$conf.int[1] * 100), "%, ",
    round(binom.test(600, 1000)$conf.int[2] * 100), "%]\n")

10.3 列联表回答的是哪一种独立性问题

列联表（contingency table）交叉分类两个分类变量。对于森林防治数据，我们关心：处理（对照/综合防治）和受害状态（是/否）是否独立？

受害 未受害 合计
对照 a b a+b
综合防治 c d c+d
合计 a+c b+d n

零假设：处理与受害状态独立：即受害比例在处理间无差异。

但注意：这个表里的计数应该以小区为单位还是以树木为单位？如果以树木为单位（a = 对照组受害树木数），我们假设了每棵树是独立的。这违反了树木嵌套于小区的设计。正确做法是先在小区层级汇总（见 10.6 节）。

10.4 卡方与 Fisher 方法的条件

卡方检验（χ² test）：基于大样本正态近似。适用条件：期望频数 ≥ 5（传统标准）。当期望频数太小时，卡方近似不可靠。

Fisher 精确检验：基于超几何分布，不依赖大样本近似。适用于小样本或稀疏表格。但 Fisher 检验假设行列边际固定。这在某些设计下不完全合理。

# 先在小区层级汇总：以小区是否超过受害阈值为分类
# 定义：小区受害率 > 30% 为"高受害"
plot_damage$high_damage <- plot_damage$prop > 0.3

# 只取对照和综合防治
pd2 <- plot_damage[plot_damage$treatment %in% c("control", "integrated"), ]

# 列联表
tbl <- table(pd2$treatment, pd2$high_damage)
print(tbl)

# 卡方检验
chisq.test(tbl)

# Fisher 精确检验（适用于小样本）
fisher.test(tbl)

10.5 风险差、风险比与优势比

三种效应量回答不同的问题：

风险差（risk difference, RD）：对照风险 − 处理风险。最直观：“综合防治使受害率降低了 X 个百分点”。单位是百分点。

风险比（risk ratio, RR）：处理风险 / 对照风险。“综合防治组的受害风险是对照组的 X 倍”。

优势比（odds ratio, OR）：处理组的优势 / 对照组的优势。优势 = 受害概率 / (1 − 受害概率)。OR 在数学上方便（与逻辑回归直接对应），但在解释上容易误导：当基线风险不低时，OR 会严重偏离 RR。

# 计算三种效应量
# 从列联表
a <- tbl["control", "TRUE"]    # 对照高受害
b <- tbl["control", "FALSE"]   # 对照低受害
c <- tbl["integrated", "TRUE"] # 综合防治高受害
d <- tbl["integrated", "FALSE"]# 综合防治低受害

# 风险
risk_ctrl <- a / (a + b)
risk_int <- c / (c + d)

# 风险差
rd <- risk_ctrl - risk_int
cat("风险差 (RD):", round(rd * 100, 1), "个百分点\n")

# 风险比
rr <- risk_int / risk_ctrl
cat("风险比 (RR):", round(rr, 2), "\n")

# 优势比
odds_ctrl <- risk_ctrl / (1 - risk_ctrl)
odds_int <- risk_int / (1 - risk_int)
or <- odds_int / odds_ctrl
cat("优势比 (OR):", round(or, 2), "\n")

cat("\n注意：当基线风险为", round(risk_ctrl * 100), "%时，\n")
cat("OR =", round(or, 2), "不等于 RR =", round(rr, 2), "\n")
黑豹提醒：优势比不是风险比

当基线风险为 50% 时，OR = 2.0 对应的 RR 只有约 1.33。报告”风险降低 50%“（OR = 0.5）而实际绝对风险只从 50% 降到 33%。这在保护沟通中可能严重误导管理决策。

10.6 聚集、稀疏与完全分离

分类数据在保护研究中有两个特殊问题：

聚集（clustering）：树木嵌套于小区。同一小区的树木共享处理和环境，不是独立的。把树木汇总成 2×2 表（忽略小区）等于制造伪重复。

完全分离（complete separation）：当某个处理的观测全部是同一结果时（如综合防治所有小区都是低受害），标准 logistic 回归无法拟合。这是第 12 章的接口。

# 反例：树级汇总（忽略小区聚集）
tree_tbl <- table(f3$treatment[f3$treatment %in% c("control", "integrated")],
                  f3$damaged[f3$treatment %in% c("control", "integrated")])
print(tree_tbl)
cat("树级卡方 P =", round(chisq.test(tree_tbl)$p.value, 4), "\n")

# 正确做法：小区级汇总
cat("小区级 Fisher P =", round(fisher.test(tbl)$p.value, 4), "\n")

cat("\n树级分析把 400 棵树当作独立观测：\n")
cat("实际上只有 20 个独立实验单位（小区）。\n")
cat("树级 P 值通常更小，不是因为效应更强，而是因为伪重复。\n")

10.7 从比例证据到风险沟通

完整的风险沟通必须包含： 1. 基线风险：对照组受害率是多少？ 2. 绝对风险降低：综合防治使受害率降低了多少个百分点？ 3. 相对风险降低：降低的比例是多少？ 4. 不确定性。这些估计的置信区间？ 5. 聚集提醒：分析在小区层级进行，树木不是独立重复。

cat("=== 风险沟通摘要 ===\n\n")
cat("对照组高受害小区比例:", round(risk_ctrl * 100), "%\n")
cat("综合防治组高受害小区比例:", round(risk_int * 100), "%\n")
cat("绝对风险降低:", round(rd * 100), "个百分点\n")
cat("相对风险:", round(rr, 2), "\n")
cat("\n注意：分析在小区层级（n = 20 个小区），不是树木层级。\n")
cat("未纳入分析：非目标影响、防治成本、实施完整性。\n")

本章小结

  • 比例必须保留分子和分母。3/5 和 60/100 都是 60%，但证据强度完全不同。
  • 卡方检验依赖大样本近似；Fisher 检验适用于小样本。选择取决于期望频数，不是”哪个 P 值更小”。
  • 优势比不是风险比。当基线风险不低时，OR 会严重偏离 RR：在保护沟通中优先使用风险差和风险比。
  • 树木嵌套于小区：树级汇总制造伪重复。分类分析同样需要尊重设计层级。
  • 风险沟通必须同时给出基线风险和绝对差异。只报告”风险降低 50%“而不说基线风险是 2% 还是 50%，是误导。

练习

  1. 识别层。以下比例的分母是什么？证据强度如何？

    1. “发芽率 80%”（20 粒种子，16 粒发芽）
    2. “检出率 45%”（60 个样点，27 个至少一次检出）

  2. 解释层。为什么优势比不等于风险比？在什么情况下两者接近？

  3. 应用层。用森林数据比较标准防治和生物防治的”高受害”比例。使用小区级分析，报告风险差和风险比。

  4. 诊断层。树级卡方检验 P = 0.001，小区级 Fisher 检验 P = 0.08。为什么差异这么大？哪个更可信？

  5. 综合层。把相机检出数据（detected）写成二项结构：分子是什么？分母是什么？独立单位是什么？为什么不能把 1080 行当作独立二项试验？
