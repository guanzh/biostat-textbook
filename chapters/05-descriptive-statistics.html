→
③ 形成画像

5.1 描述之前，先问每一行代表什么

第 4 章告诉你数据表的主键锁定”一行代表什么”。描述统计的第一步，就是把这个认识带到计算里。

本章的主项目是栖息地修复监测（项目 D）。6 个景观区组中，每个景观有一块修复地和一块对照地，在修复前和修复后各调查 5 个样方，共 120 行数据。响应变量包括冠层盖度、土壤湿度、物种丰富度和本地物种盖度。

在开始计算任何均值之前，先回答三个问题：

  1. 每一行代表什么？ 一个样方 × 一个时期的一次调查。120 行 = 6 景观 × 2 地点 × 2 时期 × 5 样方。
  2. 哪些行之间不是独立的？ 同一个地点内的 5 个样方共享该地点的所有条件（土壤基底、水文、管理历史）。同一个景观内的修复地和对照地共享景观尺度的气候和干扰历史。
  3. “样本量”应该数什么？ 如果你在比较修复地和对照地的差异，独立单位是地点（site_id，n = 12），不是样方（n = 120）。

如果你跳过这三个问题，直接把 120 行数据丢进 mean() 和 sd()，你会得到一些数字——但它们回答的问题可能和你以为的完全不同。

# 读取修复地数据
rest <- read.csv("../data/teaching/restoration_site_period.csv",
                 stringsAsFactors = FALSE)

# 先看结构
str(rest)

# 检查设计层级
table(rest$landscape_id, rest$status, rest$period)
# 每个景观 × 地点类型 × 时期 应有 5 个样方

观察问题： 修复地物种丰富度的均值是 14.2。这个数字是对什么问题的回答？是对”修复地样方的平均丰富度”还是”修复措施的平均效果”？两者的区别在哪里？

5.2 用原始分布而不是一个均值认识数据

均值是最常用的摘要量，也是最容易被滥用的。两组数据可以有完全相同的均值和标准差，但分布形状完全不同。

反例： 下面两组数据具有相同的均值（10）和标准差（约 4.6），但它们的生态含义截然不同：

  • 组 A：8, 9, 10, 11, 12——对称分布，每个值都接近中心。
  • 组 B：0, 0, 10, 20, 20——大量零值和少量极高值，反映的是一个”要么没有、要么很多”的生态过程。

在保护生物学中，组 B 的模式非常常见：很多样方没有目标物种（零值），少数样方有大量个体（聚集分布）。如果只看均值和标准差，你会以为这是一个”围绕 10 均匀波动”的系统——但实际上它是一个”大部分地方没有、少数地方爆发”的系统。这两种系统的管理策略完全不同。

# 构造反例：相同均值，不同分布
set.seed(20260620)
group_A <- c(8, 9, 10, 11, 12)
group_B <- c(0, 0, 10, 20, 20)

# 均值和标准差完全相同
mean(group_A); sd(group_A)
mean(group_B); sd(group_B)

# 但分布完全不同
# 画原始点
stripchart(list(A = group_A, B = group_B), method = "jitter",
           vertical = TRUE, pch = 16, col = c("#4d8294", "#c86652"),
           main = "相同均值，不同分布")

认识数据的正确顺序是：先看原始值，再看分布形状，最后才计算摘要量。 不要反过来——不要先算均值，然后用均值去”想象”数据长什么样。

# 修复地物种丰富度：先看原始分布
par(mfrow = c(1, 2))
hist(rest$species_richness, breaks = 15, col = "#e1f5fe",
     main = "物种丰富度分布", xlab = "物种数")
# 按修复状态分组看
boxplot(species_richness ~ status, data = rest,
        col = c("#ffebee", "#e8f5e9"),
        names = c("对照", "修复"), ylab = "物种丰富度")

5.3 集中趋势回答什么，离散程度回答什么

描述统计的两大类摘要量回答的是不同的问题：

摘要量 回答的问题 受极端值影响
均值（mean） “如果所有观测值平均分配，每个单位得到多少？” 强
中位数（median） “一半的观测值在它之上，一半在它之下——典型值是多少？” 弱
标准差（SD） “典型的一个观测值偏离均值多远？” 强
四分位距（IQR） “中间 50% 的观测值跨越多大范围？” 弱
范围（range） “最小到最大跨越多大？” 极强

选择哪个摘要量，取决于你的数据特征和你真正想问的问题。如果数据对称且没有极端值，均值和中位数接近，用哪个都可以。如果数据偏态或有极端值，中位数和 IQR 更能反映”典型”情况。

# 修复地物种丰富度的摘要量
summary(rest$species_richness)
# 均值
mean(rest$species_richness)
# 中位数
median(rest$species_richness)
# 标准差
sd(rest$species_richness)
# 四分位距
IQR(rest$species_richness)
# 范围
range(rest$species_richness)

关键判断： 均值和中位数回答不同的问题。当数据偏态时，用均值描述”典型值”会误导——均值被少数极端值拉向尾部，不再代表大多数观测值的位置。

5.4 偏态、零值、界限与离群点

保护数据有四个常见的分布特征，每一个都必须在建模前被识别：

偏态（skewness）： 分布不对称，一条尾巴比另一条长。例如大多数样方的虫口密度很低，少数样方极高——这是典型的右偏分布。右偏数据的均值大于中位数。

零值（zero-inflation）： 很多观测值为零，超过了标准分布（如 Poisson）能解释的范围。例如大部分红外相机 occasion 没有拍到目标物种。

界限（bounds）： 变量有天然的上下限。物种丰富度 ≥ 0（下界），冠层盖度在 0–100% 之间（上下界）。这些界限意味着方差在边界附近会缩小——你不能在 100% 盖度的样方上再增加 20% 的盖度。

离群点（outliers）： 远离大多数观测值的个别点。离群点首先应该被核查——是录入错误？是仪器故障？还是真实的极端事件？只有在确认是错误后才能修正或排除；如果它是真实的，它可能恰恰是你研究中最有信息量的观测。

# 检查冠层盖度的分布特征
hist(rest$canopy_cover, breaks = 20, col = "#e8f5e9",
     main = "冠层盖度分布", xlab = "盖度 (%)")
abline(v = mean(rest$canopy_cover), col = "#c86652", lwd = 2, lty = 2)
abline(v = median(rest$canopy_cover), col = "#4d8294", lwd = 2, lty = 2)
legend("topright", legend = c("均值", "中位数"),
       col = c("#c86652", "#4d8294"), lty = 2, lwd = 2)

# 检查是否有离群点（用箱线图的标准）
bp <- boxplot(rest$canopy_cover, plot = FALSE)
# 标记离群点但不删除
rest$canopy_outlier <- rest$canopy_cover %in% bp$out
table(rest$canopy_outlier)
黑豹提醒：离群点先核查，不能自动删除

离群点可能是录入错误（“100”打成了”1000”），也可能是你研究中最重要的观测——那棵在极端干旱中存活下来的树、那个唯一拍到目标物种的样点。在确认来源之前，标记但不删除。

5.5 分组与层级会改变总体描述

把数据合并在一起算一个总均值，是描述统计中最常见的错误之一。修复地数据天然有四个分组维度：

  • 时期： 修复前 vs 修复后
  • 地点类型： 修复地 vs 对照地
  • 景观区组： 6 个景观
  • 样方： 嵌套于地点内

如果你把 120 行数据合并在一起算”总均值”，你实际上是在回答一个没有人问的问题：“如果不分修复前后、不分修复对照、不分景观差异，所有样方的平均丰富度是多少？”这个数字对于评估修复效果没有任何用处——它把修复效应、时间趋势和景观差异全部搅在一起了。

# 分层描述：按时期和地点类型
aggregate(species_richness ~ period + status, data = rest, FUN = mean)
aggregate(species_richness ~ period + status, data = rest, FUN = sd)

# 按景观区组看异质性
aggregate(species_richness ~ landscape_id, data = rest, FUN = mean)

# 错误做法：合并所有分组
mean(rest$species_richness)  # 这个数字回答的是什么问题？

正确的做法是：先分层，再描述。 每一层的摘要量只回答该层的问题。修复后的修复地样方丰富度均值，只描述”修复地样方在修复后的状态”——它不自动等于”修复措施的效果”。效果需要对照地和修复前的数据来估计（第 16 章）。

# 按景观和时期画分组箱线图
par(mfrow = c(1, 2))
# 修复前
boxplot(species_richness ~ status,
        data = rest[rest$period == "before", ],
        col = c("#ffebee", "#e8f5e9"),
        main = "修复前", ylab = "物种丰富度",
        names = c("对照", "修复"))
# 修复后
boxplot(species_richness ~ status,
        data = rest[rest$period == "after", ],
        col = c("#ffebee", "#e8f5e9"),
        main = "修复后", ylab = "物种丰富度",
        names = c("对照", "修复"))

关键判断： 合并分组会掩盖交互效应。修复地丰富度从 12 升到 16，对照地从 12 升到 13——如果你只看”修复后所有样方的均值”，你会把时间趋势误认为修复效果。

5.6 正态分布是模型假设，不是数据目标

很多教材的第一句话是”正态分布是统计学中最重要的分布”。这句话本身没错，但它被误解的方式造成了大量伤害。

正态分布是某些统计模型的假设——t 检验、ANOVA、线性回归的残差被假定来自正态分布。但它不是数据的天然目标。你的数据不需要”是正态的”才能被分析——它只需要被正确的模型处理。

保护数据中常见的非正态模式：

  • 计数数据（物种数、虫口数、照片数）→ Poisson 或负二项分布（第 12 章）
  • 0/1 数据（检出/未检出、存活/死亡）→ 二项分布（第 10、12 章）
  • 比例数据（盖度、死亡率）→ Beta 分布或二项 GLM（第 12 章）
  • 右偏连续数据（生物量、密度）→ 对数正态或 Gamma 分布

“先做正态性检验，不显著就用 t 检验，显著就改用非参数检验”——这是教科书里常见的流程，但它是错的。问题不在于”数据是不是正态”，而在于”数据的生成过程应该用什么分布来描述”。计数数据不会因为”正态性检验不显著”就变成连续数据。

# 演示：丰富度数据的分布
# 丰富度是计数数据（非负整数），天然不适合用正态分布描述
hist(rest$species_richness, breaks = 15, col = "#e1f5fe",
     main = "物种丰富度：计数数据，不是正态数据",
     xlab = "物种数")

# 叠加一个拟合的正态曲线——注意它延伸到负数区域
curve(dnorm(x, mean = mean(rest$species_richness),
            sd = sd(rest$species_richness)) * nrow(rest) * 2,
      add = TRUE, col = "#c86652", lwd = 2)
# 正态曲线延伸到 x < 0——但丰富度不可能为负

关键判断： 正态分布是某些模型的残差假设，不是所有数据的天然形态。不要为了”让数据正态”而对数据做变换——选择适合数据生成过程的模型，比扭曲数据去适应模型更可靠。

5.7 写出分析前数据画像

在进入任何正式模型之前，你应该能在一页纸上说清以下内容。这份文档叫”分析前数据画像”（pre-analysis data profile），它的作用是：在模型假设和数据现实之间做第一次对照。

数据画像的固定栏目：

栏目 内容
研究问题 一句话：我们要判断什么？
数据来源 文件名、设计、独立单位、观测单位
响应变量 名称、类型、单位、取值范围、缺失情况
分组变量 处理因素、区组、时间、空间层级
分布特征 中心、离散、偏态、零值比例、界限、可疑离群点
层级提醒 哪些观测共享同一个上层单位？哪些分组不应被合并？
模型约束 基于以上观察，哪些模型假设可能被违反？哪些模型更适合？ 
# 为修复地物种丰富度写数据画像的关键数字
cat("=== 修复地物种丰富度数据画像 ===\n")
cat("观测单位：样方 × 时期，共", nrow(rest), "行\n")
cat("独立单位：地点（site_id），共",
    length(unique(rest$site_id)), "个\n")
cat("分组：", length(unique(rest$landscape_id)), "个景观 × 2 种地点 × 2 个时期\n\n")

cat("--- 总体 ---\n")
cat("均值:", round(mean(rest$species_richness), 1),
    "中位数:", median(rest$species_richness),
    "SD:", round(sd(rest$species_richness), 1),
    "IQR:", IQR(rest$species_richness),
    "范围:", paste(range(rest$species_richness), collapse = "–"), "\n")

cat("\n--- 按时期和地点类型 ---\n")
print(aggregate(species_richness ~ period + status, data = rest,
                FUN = function(x) c(mean = round(mean(x), 1),
                                    sd = round(sd(x), 1))))

cat("\n--- 零值比例 ---\n")
cat("丰富度 = 0 的样方:", sum(rest$species_richness == 0),
    "(", round(100 * mean(rest$species_richness == 0), 1), "%)\n")

cat("\n--- 模型约束 ---\n")
cat("丰富度是计数数据（非负整数），不应直接用正态线性模型。\n")
cat("样方嵌套于地点，不应把 120 行当作独立观测。\n")
cat("修复效果需要对照地和修复前数据共同估计（BACI，第 16 章）。\n")

本章小结

  • 描述统计是正式建模前对数据生成过程的第一次审问。 先看原始值和分布，再算摘要量——不要反过来。
  • 均值和标准差不是唯一答案。 中位数和四分位距在偏态数据中更能反映”典型”情况；选择摘要量取决于你的数据特征和你想回答的问题。
  • 相同的摘要量可以对应完全不同的分布。 两组均值和标准差相同的数据，可能一组对称、另一组由大量零值和少量极高值构成——生态含义完全不同。
  • 合并分组会掩盖交互效应。 修复地数据有时期、地点类型、景观和样方四个层级——每一层的摘要量只回答该层的问题。
  • 正态分布是模型假设，不是数据目标。 计数、0/1、比例和右偏数据需要对应的分布模型，不要为了”让数据正态”而扭曲数据。
  • 分析前数据画像让你在建模之前就识别出潜在的假设违反。 一页纸说清单位、分组、分布、异常和模型约束。

练习

  1. 识别层。 以下场景中，应该用均值还是中位数来描述”典型值”？为什么？

    1. 60 个红外相机样点的独立照片数（多数样点 0–5 张，少数样点 > 50 张）
    2. 32 个黄黄草小区的株高均值

  2. 解释层。 “这两组的均值和标准差几乎一样，所以它们的生态过程也应该一样。” 请用本章的反例（组 A vs 组 B）说明为什么这个推理不成立。

  3. 应用层。 修复地数据中，有研究者发现一个样方的冠层盖度为 98%，远高于其他样方。请写出你应该做的三步核查流程。

  4. 诊断层。 有人把修复前后的所有样方数据合并，计算”总均值 = 14.2”，然后报告”修复地物种丰富度为 14.2 种”。这个数字至少混合了哪三种不同的变异来源？

  5. 综合层。 为森林病虫害数据的虫口密度（pest_count）写一份分析前数据画像。包含：观测单位、独立单位、分组变量、分布特征（偏态、零值比例、离群点）、以及基于观察的模型约束建议。
