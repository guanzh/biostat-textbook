13.1 防治行动要改变什么

第三篇教你掌握了从两组比较到 GLM 的统计方法。第四篇把这些方法放回真实的保护场景，不是再讲一遍方法，而是教你如何把保护问题写成完整的研究方案。

本章聚焦于森林病虫害防治试验（项目 C）。假设你是一名森林保护技术员，负责评估四种防治措施的效果：对照（不处理）、标准化学防治、生物防治、综合防治（化学+生物+营林）。

在开始设计试验之前，先明确：防治行动要改变什么？

  • 主要靶标：虫口密度（pest_count）：防治措施的直接目标
  • 次要响应：受害等级（damage_class）、树木死亡率（dead）
  • 非目标影响：天敌和其他非目标生物的数量（non_target_count）
  • 成本：每小区的防治成本（cost_cny_plot）
  • 实施质量：措施是否按方案执行（implementation_score）

不能只看虫口密度下降了就宣布成功。如果生物防治把害虫压下去了但把天敌也杀光了，长期来看可能更糟。

forest <- read.csv("../data/teaching/forest_pest_trial_long.csv",
                   stringsAsFactors = FALSE)
定义行动靶标与副作用
划分层级区组→小区→树木
控制污染缓冲与边缘
测量多结果虫害/死亡/非目标/成本
连接决策阈值与行动

13.2 林分、样地与树木层级

森林试验的设计层级：

  • 林分（stand）：试验所在的整个森林区域：目标总体
  • 区组（block）：10 个区组，按地形和林分异质性划分：控制已知差异
  • 小区（plot）：40 个处理小区（每区组 4 个）：实验单位，处理施加在此层
  • 树木（tree）：每小区 20 株：观测单位，不是实验单位
  • 调查次数（visit）：3 次重复调查：重复测量，不是独立观测
# 检查设计
cat("区组数:", length(unique(forest$block_id)), "\n")
cat("小区数:", length(unique(forest$plot_id)), "\n")
cat("处理:", unique(forest$treatment), "\n")

# 每个区组 × 处理的树木数
table(forest$block_id, forest$treatment)

13.3 区组、缓冲与处理污染

区组：10 个区组按地形（坡位、坡向）和林分（树种组成、龄级）划分。区组内的 4 个小区条件尽可能接近：区组间的差异从分析中扣除。

缓冲与处理污染：相邻小区之间需要缓冲带，防止： - 药剂漂移（化学防治飘到生物防治小区） - 天敌迁移（生物防治小区的天敌扩散到对照小区） - 边缘效应（小区边缘的树木受相邻处理影响）

反例：“四片林地各用一种处理，再把数百株树当重复。”。这不是随机区组设计。四片林地的地形、土壤、林分历史可能完全不同：处理效应和地块效应完全混杂。

13.4 虫害、林木和非目标影响

森林防治试验有多个响应变量，每个需要不同的分析方法：

响应 类型 分析接口
虫口密度（pest_count） 计数 第 9 章 ANOVA（小区聚合）或第 12 章 GLM
受害等级（damage_class） 有序分类 0-4 第 10 章分类分析
死亡（dead） 0/1 第 10 章分类分析 / 第 12 章逻辑回归
非目标生物（non_target_count） 计数 第 12 章 GLM
成本（cost_cny_plot） 连续 与效果联合分析（第 16 章）

不能把多指标随意合成总分。虫口密度降了但树木死亡率没降。这两个指标回答不同的问题，不能加权平均成一个”综合效果分”。

# 各处理第三次调查的响应汇总
f3 <- forest[forest$visit == 3, ]
# 按小区聚合
plot_summary <- aggregate(
  cbind(pest_count, damage_class, dead, non_target_count) ~
    block_id + plot_id + treatment,
  data = f3, FUN = mean
)
# 按处理汇总
aggregate(cbind(pest_count, dead, non_target_count) ~ treatment,
          data = plot_summary, FUN = mean)

13.5 重复调查与质控

三次调查（visit 1, 2, 3）不是三次独立试验。同一小区的树木被重复测量。这引入了时间维度的设计考量：

  • 防治效果的时滞：第一次调查时效果可能尚未显现：主要分析应聚焦第三次调查
  • 调查员一致性：三次调查应由同一调查员或经过一致性培训的调查员完成
  • 设备校准。如果使用仪器测量，每次调查前校准

13.6 效果、成本和阈值

防治决策不能只看”是否统计显著”。一个完整的行动阈值表应包含：

维度 问题 本章能回答？
效果大小 虫口密度降低了多少？ 是（第 8、9 章）
效果不确定性 置信区间有多宽？ 是
非目标影响 天敌是否受损？ 是（描述性）
成本 每公顷多少钱？ 是（固定值）
实施完整性 方案是否被正确执行？ 是（implementation_score）
长期效果 效果是否持续？ 否（需要多年数据）
抗药性风险 害虫是否会产生抗性？ 否（需要专门设计） 
# 效果-成本初步分析
plot_summary$cost <- forest$cost_cny_plot[
  match(plot_summary$plot_id, forest$plot_id)]

# 各处理的平均效果和成本
trt_eval <- aggregate(
  cbind(pest_count, cost) ~ treatment,
  data = plot_summary,
  FUN = mean
)
trt_eval$cost <- trt_eval$cost / 20  # 每株树的成本
print(trt_eval)

13.7 分析接口

本章不重复讲授完整的 ANOVA 或 GLM。这些方法已在第 8-12 章中详细展开。本章的角色是把防治问题映射到正确的分析方法：

分析问题 推荐方法 章节
四种处理的虫口密度比较 RCBD ANOVA（小区聚合） 第 9 章
对照 vs 综合防治 区组内配对 t 检验 第 8 章
受害比例比较 卡方/Fisher + 风险差 第 10 章
虫口密度的时间轨迹 重复测量 ANOVA / 混合模型 第 18 章
成本-效果权衡 描述性 + 决策框架 第 16、17 章

本章小结

  • 防治试验的设计层级：林分→区组→小区→树木。处理施加在小区，树木是观测单位。
  • 多指标不能随意合成总分。虫口、受害、死亡、非目标、成本：各回答不同问题。
  • 区组控制已知异质性，缓冲防止处理污染。四片林地各用一种处理是设计灾难。
  • 效果、成本和副作用必须同时考虑。统计显著不等于应该推广。
  • 本章不重复讲授方法。它把保护问题映射到第 8-12 章的分析接口。

练习

  1. 识别层。“四片林地各用一种处理”的设计至少有三个致命缺陷。列出它们。
  2. 解释层。为什么不能把虫口密度、受害等级和死亡率加权平均成”综合防治效果”？
  3. 应用层。为森林试验设计处理间的缓冲方案：缓冲带多宽？如何监测处理污染？
  4. 诊断层。implementation_score 为 0.6 的小区：虫口密度异常高。应该删除这个小区吗？
  5. 综合层。为森林试验写一份分析计划：每个响应变量对应什么方法、什么设计层级、什么独立单位。
