第 16 章 措施效果与保护评估

措施后变好 → 寻找反事实 → 有边界效果评估
① 定义反事实
如果没有修复，丰富度会是多少？——这个数字你永远看不到
→
② 选择对照
随机对照→BACI→匹配→DID——不同设计排除不同的替代解释
→
③ 预设行动
效果多大算成功？副作用多大算不可接受？——标准必须在看到结果前定
黑豹提醒：修复地丰富度 10→14，宣称增加 40%；同期对照 8→13——BACI 对比告诉你真相。

16.1 看不见的反事实

保护措施评估的核心难题：你永远看不到”如果没有措施会怎样”。 这个看不见的”反事实”（counterfactual）是所有效果评估的逻辑起点。

  • 修复地丰富度从 10 升到 14 → 增加了 4 种
  • 但如果没有修复，丰富度可能从 10 升到 13（自然恢复趋势）
  • 修复的真实效果 = (14 − 10) − (13 − 10) = 1 种——不是 4 种

没有对照组的”前后比较”隐含了一个强假设：没有措施的情况下，结果不会变化。 在生态系统中，这个假设几乎从不成立——植被会自然演替、气候会波动、动物会迁移。

rest <- read.csv("../data/teaching/restoration_site_period.csv",
                 stringsAsFactors = FALSE)
定义反事实看不见的对照
选择对照随机/BACI/匹配
计算交互变化之差
检查破坏平行趋势/溢出/失访
预设行动成功标准与停止规则

16.2 随机与观察性评估边界

效果评估设计从强到弱：

  1. 随机对照试验（RCT）： 随机分配处理/对照——最强因果推断。森林病虫害试验属于此类。
  2. BACI（Before-After-Control-Impact）： 处理前后 + 对照——控制共同时间趋势。修复地监测属于此类。
  3. 匹配（Matching）： 为每个处理单元找一个”相似”的对照单元——只平衡已测变量。
  4. 差异中的差异（DID）： 比较处理组和对照组在政策前后的变化差——假设平行趋势。
  5. 单纯前后比较： 没有对照组——最弱，几乎不能排除任何替代解释。

随机化增强可比性，但不保证可比性。 即使随机分配，小样本下两组基线仍可能偶然不同——这就是为什么需要检查基线平衡。

16.3 单纯前后比较为何不够

反例（本章核心反例）： “修复地丰富度从 10 升到 14，增加了 40%。”

这个结论假设了：如果没有修复，丰富度会保持在 10。但同期对照地从 8 升到 13——说明存在一个区域性的丰富度上升趋势（可能是降水增加、气温变化、或其他未测量的因素）。

# 计算各组前后变化
rest_agg <- aggregate(
  species_richness ~ landscape_id + site_id + status + period,
  data = rest, FUN = mean
)

# 前后变化
rest_before <- rest_agg[rest_agg$period == "before", ]
rest_after <- rest_agg[rest_agg$period == "after", ]

# 修复地变化
restored_change <- rest_after$species_richness[rest_after$status == "restored"] -
  rest_before$species_richness[rest_before$status == "restored"]
# 对照地变化
control_change <- rest_after$species_richness[rest_after$status == "control"] -
  rest_before$species_richness[rest_before$status == "control"]

cat("修复地平均变化:", round(mean(restored_change), 1), "\n")
cat("对照地平均变化:", round(mean(control_change), 1), "\n")
cat("BACI 效果（变化之差）:", round(mean(restored_change - control_change), 1), "\n")

16.4 BACI 的交互差异

BACI 的核心是处理 × 时期的交互作用。统计模型：

Y = β₀ + β₁×Status + β₂×Period + β₃×Status×Period + ε

  • β₁：修复前的处理-对照差异（基线不平衡）
  • β₂：对照组的时期变化（共同时间趋势）
  • β₃：BACI 效果——修复的额外变化（我们关心的）
# 拟合 BACI 模型（简化版：按地点聚合后）
rest_site <- aggregate(
  species_richness ~ landscape_id + site_id + status + period,
  data = rest, FUN = mean
)

rest_site$status <- factor(rest_site$status)
rest_site$period <- factor(rest_site$period)

model_baci <- lm(species_richness ~ status * period, data = rest_site)
summary(model_baci)

# BACI 效果
cat("\nBACI 效果（statusrestored:periodafter）:\n")
cat("估计:", round(coef(model_baci)["statusrestored:periodafter"], 2), "\n")
cat("95% CI: [",
    round(confint(model_baci)["statusrestored:periodafter", 1], 2), ",",
    round(confint(model_baci)["statusrestored:periodafter", 2], 2), "]\n")

16.5 匹配、差异中的差异与假设

BACI 的强假设：平行趋势——如果没有修复，修复地和对照地的变化趋势相同。这个假设无法直接检验（因为你永远看不到修复地的反事实），但可以通过以下方式间接评估：

  • 修复前两组的轨迹是否平行？
  • 对照地是否受到修复的间接影响（溢出）？
  • 是否有其他事件在修复前后同时发生（混杂）？

匹配可以增强可比性，但只平衡已测量的变量。如果未测量的变量同时影响处理分配和结果，匹配也无能为力。

16.6 实施完整性、溢出、失访和附带伤害

效果评估不只是”处理组 vs 对照组”——还需要检查：

  • 实施完整性： 修复措施是否按方案执行？如果只执行了 60%，效果打折扣是预期的。
  • 溢出（spillover）： 修复地的效果是否”溢出”到邻近的对照地？（如种子扩散）
  • 失访（attrition）： 是否有样方在修复后被破坏或无法调查？
  • 附带伤害： 修复是否带来了非预期的负面影响？（如外来种入侵）
# 检查各景观的 BACI 效果异质性
baci_by_landscape <- aggregate(
  species_richness ~ landscape_id + status + period,
  data = rest_site, FUN = mean
)

# 计算每个景观的 BACI 效果
for (l in unique(baci_by_landscape$landscape_id)) {
  ld <- baci_by_landscape[baci_by_landscape$landscape_id == l, ]
  ctrl_change <- ld$species_richness[ld$status == "control" & ld$period == "after"] -
    ld$species_richness[ld$status == "control" & ld$period == "before"]
  treat_change <- ld$species_richness[ld$status == "restored" & ld$period == "after"] -
    ld$species_richness[ld$status == "restored" & ld$period == "before"]
  cat("景观", l, "BACI 效果:", round(treat_change - ctrl_change, 1), "\n")
}

16.7 成功标准与停止规则

效果评估的最后一步：在见到结果之前，定义”成功”和”失败”。

成功标准应包含： - 主要效果： 丰富度增加 ≥ X 种（最小有意义效应） - 不确定性： 95% CI 的下限 > 0（效果不太可能为零或负） - 副作用： 非本地种比例没有显著增加 - 成本： 每增加 1 种的成本 ≤ Y 元 - 实施： 方案执行完整度 ≥ 80%

停止规则：如果中期评估显示效果远低于预期或副作用超出可接受范围——停止并重新设计方案。不能因为”已经投入了很多”而继续一个无效或有害的措施。

本章小结

  • 反事实是效果评估的逻辑起点。 你永远看不到”如果没有措施会怎样”——对照组的全部意义就是近似这个反事实。
  • 单纯前后比较几乎从不成立。 生态系统有自然趋势——没有对照组的变化不能归因于措施。
  • BACI 的核心是交互作用。 修复效果 = (修复后 − 修复前) − (对照后 − 对照前)。
  • 平行趋势是 BACI 的关键假设。 它不能被直接检验，但可以通过修复前轨迹间接评估。
  • 成功标准必须在看到结果之前定义。 效果大小、不确定性、副作用、成本——全部预先规定。

练习

  1. 识别层。 以下评估设计各属于什么类型？各能排除什么替代解释？

    1. 修复地前后比较，无对照组
    2. 修复地 + 对照地，前后各一次调查
    3. 随机分配 20 个样地，10 个修复、10 个对照

  2. 解释层。 “修复地丰富度增加 40%”——如果同期对照地增加了 30%，修复的真实效果是多少？

  3. 应用层。 用修复地数据计算每个景观的 BACI 效果。哪些景观效果最强？哪些最弱？这种异质性意味着什么？

  4. 诊断层。 修复前，修复地的平均丰富度比对照地高 3 种。这会影响 BACI 分析吗？如何影响？

  5. 综合层。 为森林防治试验写一份预先成功标准。包含：主要效果阈值、副作用上限、成本上限、以及什么情况下应该停止。
