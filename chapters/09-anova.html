① 确定单位
32 个小区是实验单位——不是 160 株幼苗
→
② 写出模型
区组 + 光照——每一项都回指第 3 章的设计
→
③ 估计对比
40% vs 100% 是主要比较——总体 F 检验只是第一步
黑豹提醒：ANOVA 显著 ≠ 每组都不同。总体检验不能替代计划对比——先决定要比较什么，再跑模型。

9.1 多组问题首先是对比问题

第 8 章比较了两组。但保护研究中经常需要同时比较三个或更多组——不同光照水平、不同防治措施、不同生境类型。ANOVA（方差分析）是处理多组比较的经典方法。

但 ANOVA 有一个致命的误解：“ANOVA 显著”不等于”每组都不同”。 F 检验只回答一个笼统的问题——“至少有一组和其他组不同”——它不告诉你哪组和哪组不同，更不告诉你差异有多大。

本章的主项目回到黄黄草光照试验（项目 A）：8 个区组 × 4 个光照水平（100%、70%、40%、15%），32 个小区，每小区 5 株幼苗，在 1、3、6 个月测量。本章聚焦于第六个月的开花比例。

在跑任何模型之前，先明确你要比较什么：

  • 主要比较： 40% 光照 vs 100% 光照——40% 对应野外林冠下典型光环境
  • 次要比较： 70% vs 100%、15% vs 100%
  • 趋势检验： 开花比例是否随光照降低而递减？

这些比较必须在看到数据之前指定——不能在看到结果后再挑”最显著”的那个。

# 读取黄黄草数据
hh <- read.csv("../data/teaching/huanghuang_light_long.csv",
               stringsAsFactors = FALSE)

# 聚焦第六个月
hh6 <- hh[hh$month == 6, ]

# 按小区汇总开花比例
plot_flower <- aggregate(
  flowered ~ block_id + plot_id + light_pct,
  data = hh6,
  FUN = mean
)
head(plot_flower)
确定单位32 个小区
写出模型区组 + 光照
分解变异可解释 vs 剩余
估计对比计划对比优先
解释边界效应、诊断、外推

学完本章后，你能够：

  1. 根据设计结构（CRD/RCBD）写出正确的 ANOVA 模型公式。
  2. 区分总体 F 检验和计划对比——前者不能替代后者。
  3. 解释交互作用如何改变主效应的含义。
  4. 进行残差诊断，识别异常小区和方差问题。
  5. 报告处理效应和置信区间，而不是只给显著字母分组。

9.2 从随机区组设计写出模型结构

第 3 章的设计直接决定了模型公式。黄黄草试验是 RCBD：8 个区组，每个区组内 4 个光照水平随机分配。

模型公式：开花比例 ~ 光照 + 区组

每一项的含义： - 光照：处理因素（固定效应）——我们关心的比较 - 区组：区组因素——控制已知的苗床南北差异

为什么区组要进模型？ 因为区组间的差异（南段 vs 北段的含水量本底）是真实存在的，但我们对它本身不感兴趣。把它放进模型，等于把区组间的变异从误差中扣除——让处理效应的检验不被区组差异污染。

# 拟合 RCBD 模型
plot_flower$light_pct <- factor(plot_flower$light_pct)
plot_flower$block_id <- factor(plot_flower$block_id)

model <- lm(flowered ~ light_pct + block_id, data = plot_flower)

# ANOVA 表
anova(model)

# 模型系数
summary(model)

9.3 方差分析如何分解可解释与剩余变异

ANOVA 的核心思想是分解总变异：

总变异（SST）= 处理间变异（SSA）+ 区组间变异（SSB）+ 剩余变异（SSE）

F 检验比较的是：处理间变异 / 剩余变异。如果处理间的变异远大于同一处理内小区间的随机变异，说明处理效应不太可能由随机波动产生。

但 F 检验的显著只是一个起点——它告诉你”至少有一组不同”，然后你必须回到预先指定的对比。

# 手动理解变异分解
# 总均值
grand_mean <- mean(plot_flower$flowered)

# 处理均值
treatment_means <- aggregate(flowered ~ light_pct, data = plot_flower, FUN = mean)

# 区组均值
block_means <- aggregate(flowered ~ block_id, data = plot_flower, FUN = mean)

cat("总均值:", round(grand_mean, 3), "\n")
cat("\n处理均值:\n")
print(treatment_means)
cat("\n区组均值:\n")
print(block_means)

9.4 总体检验不能替代计划对比

F 检验显著后，最常见的错误是：对每对处理做 t 检验，然后用显著字母（a, b, c）标注。

问题： 1. 多重比较膨胀第一类错误率。 4 组有 6 对比较——如果每对都用 α = 0.05，整体错误率远高于 0.05。 2. 显著字母不能替代效应量。 “a”和”b”不告诉你差异有多大。 3. 不是所有比较都同等重要。 40% vs 100% 是预先指定的主要比较——它应该被优先报告，不管其他比较是否显著。

计划对比（planned contrast）： 在见到数据之前指定的比较。计划对比不需要多重比较校正（或只需要对计划对比的数量做校正），因为它们不是”从所有可能比较中挑出来的”。

# 主要计划对比：40% vs 100%
library(multcomp)  # 如果不可用，用基础 R 的 contrast 方法

# 方法 1：用基础 R 的线性假设检验
# 对比系数：100% = -1, 40% = 1, 其他 = 0
contrast_matrix <- cbind(
  "40vs100" = c(-1, 0, 1, 0),  # 100%, 15%, 40%, 70%
  "70vs100" = c(-1, 0, 0, 1),
  "15vs100" = c(-1, 1, 0, 0)
)

# 使用 emmeans 包（如果可用）或手动计算
# 手动计算主要对比的估计和标准误
coefs <- coef(model)
# light_pct40 的系数相对于 light_pct100（参考水平）
# 所以 40% vs 100% 的效应就是 light_pct40 的系数
cat("40% vs 100% 效应估计：",
    round(coefs["light_pct40"], 3), "\n")

# 从模型摘要中获取
summary(model)$coefficients["light_pct40", ]

9.5 交互作用改变主效应的解释

如果试验同时操纵两个因素（如光照 × 浇水频率），就涉及交互作用。交互作用的含义是：一个因素的效应取决于另一个因素的水平。

本章不要求你拟合析因模型（那是第 18 章的内容），但你必须能识别交互作用的存在，并理解为什么”只看主效应”在交互作用存在时是错的。

简单示例：如果光照对开花的影响在浇水多时很强、在浇水少时很弱——那么”光照的平均效应”（主效应）掩盖了这种依赖关系。正确的做法是先检查交互作用是否显著，如果显著，主效应不能被单独解释。

9.6 残差、方差与独立性诊断

ANOVA 的三个核心假设： 1. 独立性： 残差之间不相关——这由设计保证（随机化） 2. 方差齐性： 各处理组的方差相近 3. 正态性： 残差近似正态分布

诊断不是”通过/不通过”的二元判断——它是帮你理解模型在哪些地方可能不可靠。

# 残差诊断
par(mfrow = c(2, 2))
plot(model)

# 按处理组检查残差
par(mfrow = c(1, 1))
boxplot(resid(model) ~ plot_flower$light_pct,
        col = c("#ffebee", "#fff8e1", "#e1f5fe", "#e8f5e9"),
        names = c("100%", "15%", "40%", "70%"),
        main = "残差按光照水平", ylab = "残差")

# 检查是否有异常小区
plot_flower$resid <- resid(model)
plot_flower$cook <- cooks.distance(model)
# 标记 Cook's D > 0.5 的小区
plot_flower[plot_flower$cook > 0.5, ]
黑豹提醒：诊断不是”通过/不通过”

残差图显示一个小区异常，不意味着你应该删除它——先核查：录入错误？遮阴帘破损？还是真实的极端响应？只有在确认是错误后才能修正；如果是真实响应，保留它并做敏感性分析（有它和没它的结论是否一致）。

9.7 从光照对比到微生境建议

回到保护问题：林冠郁闭度增加是否抑制黄黄草开花？

# 完整分析报告
cat("=== 黄黄草光照试验：第六个月开花比例 ===\n\n")

# 处理均值
cat("处理均值:\n")
print(aggregate(flowered ~ light_pct, data = plot_flower,
                FUN = function(x) c(mean = round(mean(x), 3),
                                    sd = round(sd(x), 3))))

# 总体 F 检验
cat("\n总体 F 检验:\n")
print(anova(model))

# 主要对比：40% vs 100%
cat("\n主要对比（40% vs 100%）:\n")
cat("效应估计:", round(coef(model)["light_pct40"], 3), "\n")
ci_40 <- confint(model)["light_pct40", ]
cat("95% CI: [", round(ci_40[1], 3), ",", round(ci_40[2], 3), "]\n")

# 趋势：开花比例是否随光照递减？
# 将光照转为数值做趋势检验
plot_flower$light_num <- as.numeric(as.character(plot_flower$light_pct))
model_trend <- lm(flowered ~ light_num + block_id, data = plot_flower)
cat("\n线性趋势检验:\n")
cat("每降低 1% 光照，开花比例变化:",
    round(coef(model_trend)["light_num"], 4), "\n")
cat("趋势 P 值:",
    round(summary(model_trend)$coefficients["light_num", "Pr(>|t|)"], 4), "\n")

# 结论边界
cat("\n=== 结论边界 ===\n")
cat("支持：40% 光照下开花比例显著低于 100% 光照。\n")
cat("不支持：所有光照水平两两不同。\n")
cat("不支持：野外成体的开花响应与苗圃幼苗相同——外推有损耗。\n")
cat("未分析：光照对成活率、株高和 SPAD 的影响（次要响应）。\n")

关键判断： ANOVA 是”按设计分解变异并估计预先关心的对比”——不是”三组以上的 t 检验”。总体 F 检验只是第一步，计划对比才是回答研究问题的核心。

本章小结

  • ANOVA 不是”三组以上的 t 检验”。 它是按设计分解变异，并估计预先指定的对比。
  • 总体 F 检验不能替代计划对比。 “ANOVA 显著”只告诉你至少有一组不同——你必须回到预先指定的比较。
  • 区组因素必须进入模型。 区组间的差异如果不被扣除，会污染处理效应的检验。
  • 交互作用改变主效应的解释。 如果交互显著，主效应不能被单独解释。
  • 诊断不是”通过/不通过”。 异常小区先核查来源，再做敏感性分析——不能自动删除。
  • 显著字母不能替代效应图。 报告处理效应和置信区间，而不是只给 a、b、c 标注。
