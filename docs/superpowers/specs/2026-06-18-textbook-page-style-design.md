# 《从样地到结论》教材页面设计系统规范

> 状态：已确认设计方向，待全书实施
>
> 适用范围：本项目所有 Quarto 章节页面、章首插图、教学图表与练习区域
>
> 设计方向：清爽自然手账（Clean Field Notebook）

## 1. 目的

本规范用于确保不同作者和不同 AI 在编写、改写或排版本教材时，能够产出统一、舒适且具有辨识度的页面。页面应让学生感到自己正在阅读一本现代的自然观察教材，而不是浏览营销网站，也不是面对一整页未经设计的讲义。

设计的核心比例是：**80% 安静、清晰的阅读区域，20% 有教学作用的手账元素。** 手账感用于建立情境、标记重点和解释数据；正文区域始终以长时间阅读的舒适度为第一优先级。

## 2. 设计目标与非目标

### 设计目标

- 让学生在电脑网页上连续阅读 30 至 60 分钟仍然舒适。
- 打开任意章节时，能够立即识别教材品牌、当前主题和学习路线。
- 使用插图、图表和黑豹向导帮助理解，而不是单纯装饰页面。
- 让 22 章共享同一套视觉语言，避免每章临时发明新样式。
- 保持 Quarto 内容文件清晰，正文与视觉样式职责分离。

### 非目标

- 不把教材设计成营销落地页或卡片式仪表盘。
- 不在每个段落旁添加插图、边框、便签或吉祥物。
- 不追求强烈的仿纸张、胶带、阴影和旋转效果。
- 不依赖在线字体、在线图片或其他在国内网络环境下不稳定的资源。
- 不用视觉效果替代概念准确性、数据证据和清晰写作。

## 3. 核心设计原则

### 3.1 正文安静

正文使用白色或接近白色的背景、受控的行宽、充足的行距和稳定的标题层级。普通段落不套卡片，不使用彩色背景，不添加装饰性边框。读者的注意力应落在句子、概念和数据上。

### 3.2 插图必须承担教学任务

章首插图负责建立保护生物学场景；正文插图负责解释对象、过程或数据关系。无法回答“这张图帮助学生理解什么”的图片，不应加入页面。

### 3.3 颜色具有固定语义

颜色不是随机装饰。全书统一使用以下含义：

- 森林绿：章节主线、核心概念、正向引导。
- 芥末黄：学习提示、观察问题、需要停下来思考的内容。
- 湖水蓝：数据、方法、测量和统计过程。
- 珊瑚红：风险、限制、误区和需要谨慎解释的内容。
- 深墨色：正文和标题。

任何信息都不能只依赖颜色表达，还必须同时使用标题、图标、边框或文字标签。

### 3.4 黑豹少而准确

黑豹是章节向导，不是每段文字的陪衬。每章原则上最多出现三次：

1. 章首场景或章节欢迎位置；
2. 一个真正重要的误区或判断提醒；
3. 章末总结或行动任务。

普通定义、一般例子和过渡段落不使用黑豹。

### 3.5 结构先于装饰

每一页首先要有清楚的阅读顺序：章节身份、核心问题、学习路线、正文、证据、判断和练习。只有在结构稳定之后，才能加入手绘线条、插图或小型批注。

## 4. 视觉设计令牌

所有颜色和尺寸应在 `styles.css` 中集中定义，不得在章节文件中重复声明。

```css
:root {
  --ink: #243129;
  --ink-muted: #59665e;
  --forest: #315e46;
  --moss: #6b906f;
  --mustard: #d0a03a;
  --lake: #4d8294;
  --coral: #c86652;
  --paper: #fcfdfc;
  --paper-soft: #f7faf8;
  --line: #d6ddd8;
  --reading-width: 720px;
  --radius: 6px;
  --space-unit: 8px;
}
```

### 字体

- 正文：系统中文无衬线字体，优先使用 `system-ui`、`Microsoft YaHei`、`PingFang SC`、`Noto Sans CJK SC`。
- 标题：与正文同一字体家族，使用字重和空间建立层级。
- 手账批注：只允许在短标签、章节编号和黑豹提醒标题中使用楷体类系统字体。
- 统计符号、代码和 R 输出：使用等宽字体。
- 不从 Google Fonts 或其他在线服务加载字体。
- 全站字间距保持为 `0`；只有全大写的短英文标签可使用轻微正字距。

### 阅读尺寸

- 桌面正文：17 至 18 px，行高 1.8 至 1.9。
- 移动端正文：16 至 17 px，行高不低于 1.75。
- 正文有效行宽：680 至 720 px。
- 中文正文每行宜保持约 30 至 38 个汉字。
- 段落间距：约 1 rem；不同概念之间使用空间，不使用多余分隔线。

### 圆角与阴影

- 圆角不得超过 8 px。
- 普通教学组件不使用厚重阴影。
- 页面级预览或章首插图可以使用非常轻的阴影。
- 不使用黑色硬阴影、强烈旋转卡片或多层嵌套卡片。

## 5. 标准章节页面结构

所有章节按以下顺序组织。内容允许增减，但不可随意改变层级关系。

### 5.1 章首区域 `chapter-opener`

章首采用紧凑的双栏结构：左侧为章节编号、标题和一句核心问题，右侧为手绘生态插图及黑豹。章首不占满整个屏幕，首屏底部必须能看到学习路线或下一部分内容。

章首包含：

- `CHAPTER XX` 和所属篇章；
- 章节标题；
- 40 至 70 字的引导语；
- 一张与章节主题直接相关的手绘生态插图；
- 黑豹可选，且不遮挡标题或主要对象。

### 5.2 学习路线 `chapter-route`

章首下方使用三至五步的水平路线，提前说明本章的认知推进。每一步包含一个短标题和不超过 12 个汉字的说明。

推荐结构：

> 观察问题 → 识别不确定性 → 选择方法 → 形成证据 → 支持行动

路线不是导航按钮，不设计成胶囊标签，也不使用大面积色块。每一步通过语义色边线区分。

### 5.3 学习目标 `learning-goals`

学习目标使用一个低对比度的提示带，不使用大型卡片。目标以可观察的学习行为表述，例如“识别、区分、解释、计算、评价”，避免只写“理解”或“掌握”。每章保留三至五项目标。

### 5.4 真实问题 `observation-question`

每章至少有一个真实的保护生物学问题作为叙事主线。问题应包含对象、场景、观测指标和需要作出的判断。案例数据应尽可能在后续图表、方法和练习中重复使用，避免正文变成互不相关的例子集合。

### 5.5 正文小节

每个小节由以下内容组成：

1. 清楚的问题式或结论式标题；
2. 一至三个完整段落；
3. 必要时加入图表、公式、代码或短例子；
4. 小节末尾给出一句承上启下的判断。

普通小节保持无框布局。不得将所有小节变成卡片，也不得在卡片中继续嵌套卡片。

### 5.6 教学图表 `evidence-figure`

图表应尽量贴近正在解释的段落。每张图必须包含图题、变量或单位、必要的样本量说明和一句解释性图注。优先展示分布、配对变化和不确定性，而不是只展示均值柱状图。

图表颜色遵守固定语义：蓝色用于观测数据和基线，珊瑚红用于对照变化或风险，绿色用于模型结果和保护主线。所有图表需要在灰度或色觉异常情况下仍能区分。

### 5.7 黑豹提醒 `mascot-note`

黑豹提醒只用于以下内容：

- 容易形成长期错误概念的统计误区；
- 会影响保护决策的重要限制；
- 必须核查的 AI 输出或分析步骤。

提醒控制在 30 至 80 字，标题具体，例如“不要只看两个均值”，不使用空泛的“温馨提示”。黑豹图像应小于提醒文本区域，不成为视觉主体。

### 5.8 关键判断 `key-judgment`

关键判断用于收束一段证据链。它采用珊瑚红虚线或左边线，不使用大色块。内容必须是一句可以独立成立的判断，例如：

> 统计学可以量化证据，但研究设计、专业知识、成本和风险共同决定是否采取行动。

### 5.9 练习与总结

章末依次出现：

1. 三至五条本章总结；
2. 从识别到评价逐步增加难度的练习；
3. 一个使用本章主案例的综合任务；
4. 可选的 AI 核查任务；
5. 下一章的自然过渡。

总结使用完整句子。练习可以编号，但不为每道题创建单独卡片。

## 6. 插图规范

### 数量

- 每章必须有一张章首手绘生态插图。
- 正文原则上增加一至两张概念插图或过程示意图。
- 图表数量由教学需要决定，不受插图数量限制。
- 黑豹每章最多出现三次。

### 视觉语言

- 使用清楚、克制的自然观察插画风格。
- 线条类似铅笔或细墨线，颜色使用本规范的绿、黄、蓝、红作为点缀。
- 背景保持白色、透明或非常浅的绿色。
- 主体必须完整可辨，不使用暗色氛围图、虚化背景或过度裁切。
- 插图中不嵌入标题、段落和复杂标签，文字由网页负责呈现。
- 不使用表情符号代替正式插图或功能图标。

### 无障碍

- 教学插图必须有说明其教学含义的替代文本。
- 纯装饰性的黑豹或线条使用空替代文本。
- 图中出现的重要关系必须在图注或正文中再次说明。

## 7. 组件语义与使用限制

| 组件 | 用途 | 推荐颜色 | 每章建议数量 |
|---|---|---:|---:|
| `chapter-opener` | 建立章节场景与身份 | 绿 | 1 |
| `chapter-route` | 展示认知路线 | 多语义色边线 | 1 |
| `learning-goals` | 展示学习目标 | 黄 | 1 |
| `observation-question` | 提出主案例问题 | 黄或无底色 | 1 至 2 |
| `evidence-figure` | 展示数据与证据 | 蓝、红、绿 | 按教学需要 |
| `mascot-note` | 提醒关键误区 | 绿或红 | 0 至 2 |
| `key-judgment` | 收束证据链 | 红 | 1 至 3 |
| `practice-block` | 组织章末练习 | 无底色或浅绿 | 1 |

同一屏幕内尽量只出现一个强调组件。连续出现多个彩色框时，应删除装饰性较弱的框或将内容还原为普通段落。

## 9. 文件职责

- `_quarto.yml`：负责全书导航、主题配置和全局渲染选项。
- `styles.css`：负责颜色令牌、排版、页面结构和全部可复用组件。
- `chapters/*.qmd`：只负责章节内容、语义结构和资源引用，不放章节专用 CSS。
- `images/chapter-openers/`：存放章首手绘插图。
- `images/figures/`：存放教学示意图和静态数据图。
- `images/mascot/`：存放黑豹向导的有限状态图。

如果某章确实需要独有的复杂组件，应先判断它是否能够服务其他章节。能够复用时加入全局组件；不能复用时再建立独立样式文件，不在正文中写大段内联 CSS。

## 10. 响应式与可访问性要求

- 主要设计视口为宽度 1280 px 的电脑网页。
- 必须同时检查宽度 390 px 的移动端，确保标题、路线、图表和代码不重叠。
- 章首双栏在窄屏上改为单栏，正文始终保持自然阅读顺序。
- 学习路线在中等宽度下变为两列，在手机上变为单列。
- 所有图片必须设置合适的替代文本。
- 正文、按钮、链接和图表文字满足清晰的颜色对比度。
- 不能仅通过悬停才能获得必要信息。
- 所有链接和交互控件应有可见焦点状态。

## 11. 禁止事项

其他 AI 执行页面设计时，不得：

- 在 `.qmd` 文件中加入大段 `<style>`；
- 给每一节、每一段或每一道练习套卡片；
- 在卡片中嵌套卡片；
- 使用大面积渐变、彩色光斑、装饰性圆球或深色背景；
- 使用巨大的营销式标题占满首屏；
- 使用超过 8 px 的圆角作为主要视觉语言；
- 随意增加新颜色、新字体或新的 callout 风格；
- 使用在线字体或未经本地保存的外部图片；
- 用表情符号代替正式插图和功能图标；
- 让黑豹遮挡标题、图表、代码或正文；
- 只展示均值而隐藏样本分布和不确定性；
- 为了视觉丰富而重复正文已经说明的信息。

## 13. 交付检查清单

每个章节完成后，执行者必须逐项确认：

### 内容结构

- [ ] 章首明确显示章节身份、核心问题和主题插图。
- [ ] 首屏仍能看到学习路线或下一部分内容。
- [ ] 学习目标使用可观察的行为动词。
- [ ] 全章围绕一个主要保护案例或问题展开。
- [ ] 标题层级连续，没有跳级。
- [ ] 重点内容没有在插图、提醒和正文中无效重复。

### 视觉一致性

- [ ] 只使用规范中的设计令牌和语义颜色。
- [ ] 普通正文没有被卡片化。
- [ ] 同一屏幕没有堆叠多个彩色强调框。
- [ ] 黑豹出现不超过三次，且每次都有明确教学作用。
- [ ] 页面没有大面积渐变、厚重阴影和过度旋转。
- [ ] 所有章节专用内联 CSS 已移除。

### 图表与插图

- [ ] 每张图都帮助解释正文中的一个具体问题。
- [ ] 图表具有图题、变量、单位、样本量或必要图注。
- [ ] 重要数据关系不只依赖颜色区分。
- [ ] 教学图片具有准确的替代文本，装饰图片使用空替代文本。

### 阅读与验证

- [ ] 桌面端正文行宽不超过 720 px。
- [ ] 1280 px 桌面视口无文字重叠、截断或布局跳动。
- [ ] 390 px 手机视口无文字重叠、截断或横向溢出。

## 14. 验收标准

当任意两名作者或两个 AI 依据本规范分别排版不同章节时，读者应能感受到它们属于同一本教材。章节可以因主题不同而使用不同的生态对象、插图和数据图，但标题层级、空间节奏、颜色含义、组件行为和黑豹的角色必须保持一致。

最终页面应同时满足三个判断：**比传统讲义更有吸引力，比课程宣传页更安静，比当前草图风更适合长期阅读。**
